# Codex 提示词模板

## 驱动开发模板

```
为 [芯片型号] 的 [外设] 设计驱动，要求：
1. 支持 [模式1] 和 [模式2]
2. 遵循 MISRA C 规范
3. 包含完整的错误处理
4. 生成单元测试代码
5. 提供使用示例
```

**示例：**
```
为 STM32F407 的 SPI1 设计驱动，要求：
1. 支持 DMA 和中断模式
2. 遵循 MISRA C 规范
3. 包含完整的错误处理
4. 生成单元测试代码
5. 提供使用示例
```

## 更详细的驱动开发模板

```
我正在开发一个嵌入式项目，使用 [芯片型号] 芯片。
需要为 [外设] 设计驱动，用于 [具体用途]。

要求：
1. 硬件配置：
   - [外设]: [引脚配置]
   - [DMA通道] 用于发送
   - [DMA通道] 用于接收
   - 中断优先级: [优先级]

2. 功能需求：
   - 初始化配置
   - 发送数据（阻塞和非阻塞）
   - 接收数据（阻塞和非阻塞）
   - 全双工通信
   - 错误处理（超时、CRC 错误等）

3. 代码规范：
   - 遵循 MISRA C:2012
   - 使用 HAL 库
   - 包含 Doxygen 注释

4. 测试要求：
   - 单元测试（使用 Unity 框架）
   - 覆盖率 ≥ 80%
   - 包含边界测试

请生成完整的驱动代码和测试代码。
```

## 中间件开发模板

```
设计一个 [组件类型]，要求：
1. 功能：[功能描述]
2. 接口：[接口要求]
3. 性能：[性能要求]
4. 测试：[测试要求]
```

**示例：**
```
设计一个轻量级消息队列，要求：
1. 功能：支持发布订阅模式
2. 接口：线程安全，支持任意数据类型
3. 性能：支持 1000+ 消息/秒
4. 测试：单元测试覆盖率 ≥ 80%
```

## 应用逻辑开发模板

```
设计一个 [组件类型]，管理 [业务逻辑]：
1. 状态：[状态列表]
2. 转换：[转换条件]
3. 动作：[转换动作]
4. 测试：[测试场景]
```

**示例：**
```
设计一个状态机，管理设备的工作模式切换：
1. 状态：空闲、运行、休眠、错误
2. 转换：按键触发、定时器触发、异常触发
3. 动作：进入/退出状态时执行相应操作
4. 测试：所有状态转换路径
```

## 批量任务模板

```
我需要同时开发 [任务1]、[任务2]、[任务3]，请帮我：
1. 设计统一的接口规范
2. 生成三个模块的代码
3. 生成测试代码
4. 生成文档
```

**示例：**
```
我需要同时开发 SPI、I2C、UART 三个驱动，请帮我：
1. 设计统一的接口规范
2. 生成三个驱动的代码
3. 生成测试代码
4. 生成文档
```
